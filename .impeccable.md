# the-binding-of-agents — Design Context

## Users

Developers using Claude Code and Codex agents who manage multiple concurrent AI sessions and want a single visual dashboard. The maintainer/primary user is a senior platform engineer who plays The Binding of Isaac. Secondary audience: TBOI fans who happen to use Claude/Codex.

Context of use: long focused work sessions, multiple agents running in parallel, switching between agents to give prompts or watch progress. The dashboard is a background view that's glanced at, not stared at — agent state should be readable in a half-second peek.

## Brand Personality

Three words: **dark, irreverent, hand-crafted**.

The project is a fan-art rebrand of an existing dashboard, framed as a love letter to TBOI's specific visual idiom. The voice is matter-of-fact technical with TBOI flavor — `boa serve`, `THE BESTIARY`, `WHO AM I?`, `REST / FIGHT / CLEAR` status pills. Never cute or apologetic. The interface should feel like discarded room of a creepy basement, not a polished SaaS product.

## Aesthetic Direction

**TBOI faithful, not TBOI-inspired.** Lift the actual game's visual language wherever possible rather than abstracting it.

Theme: **dark basement**. The product runs during focused work; bright UI is fatiguing for a glance-at-it background dashboard. Default theme `tboi-basement`, with the bestiary as a notable cream-parchment inversion (matching the in-game Bestiary book aesthetic, which uses aged paper pages).

**Palette** (`tboi-basement`):
- bg-primary: `#1a1410` (basement charcoal)
- bg-secondary: `#2a1f18` (floor brown)
- bg-tertiary: `#0e0a08` (shadow black)
- fg-primary: `#e8d9b8` (dirty cream)
- fg-secondary: `#a89876` (faded parchment)
- accent-blood: `#8a1a18` (dried blood, used for FIGHT state, errors, focused-cell borders)
- accent-soul: `#c8b85a` (soul-heart gold, used for highlights, selection rings)
- accent-tear: `#6a8aa8` (tear blue, used for REST state, links)
- border: `#3a2d22` (floor edge)
- state-rest / state-fight / state-clear: aliased to status-idle / status-error / status-done so they track theme switches

**Bestiary parchment palette** (only inside the bestiary modal):
- panel-bg: `#d8c89a` (aged cream)
- panel-border: `#3a2418` (ink brown)
- info-text: `#2a1812` (near-black on cream)
- name-text: `#5a1a18` (blood-red ink for entry names)

**Type**: `Upheaval TT BRK` (TBOI's actual UI font, free for non-commercial use, vendored as `/fonts/upheavtt.ttf`) for display; `Press Start 2P` fallback. Body text uses the same family at smaller sizes. Avoid serif body — the look is pixel-game, not editorial.

**Iconography**: 84 actual TBOI sprites (17 base characters + 17 Tainted variants + 50 familiars), sourced from `bindingofisaacrebirth.wiki.gg`, sliced to 96×96 PNG with nearest-neighbor scaling. Sprites render with `image-rendering: pixelated` — no smoothing, ever.

**Imagery**: Real TBOI room screenshot as the basement-view map background (`/basement-floor.png`, 544×480 cover-cropped from a 1920×1200 wiki screenshot). Status pills use TBOI vocabulary: REST / FIGHT / CLEAR.

**Anti-references**:
- Do NOT make it look like a polished SaaS product (Linear, Vercel, Notion) — those visuals would feel like a betrayal of the TBOI premise.
- Do NOT use gradient text, glassmorphism, soft shadows, rounded-2xl-bg-blue cards. None of TBOI uses those.
- Do NOT introduce a "modern minimal" aesthetic. The project's value is in feeling slightly cursed and hand-built, not airy and clean.
- Do NOT use stock photography or generic AI art.

## Design Principles

1. **Pixel art is sacred** — every sprite renders nearest-neighbor at integer scales. No bilinear, no anti-aliasing, no smoothing filters. Sprites slightly overflow their containers (sometimes) rather than being CSS-scaled to non-integer ratios.
2. **Theme tokens, never hex** — all color references go through the `var(--theme-*)` token system. Hex literals in component code are a code-review failure. The token system is what allows the dark-basement / cream-parchment dual aesthetic to coexist without contradiction.
3. **TBOI vocabulary first** — every user-facing string passes through a TBOI mental filter. "PC Box" is wrong; "The Bestiary" is right. "Pokégents" is wrong; "the binding of agents" is right. "Spawn agent" is correct technical English; "WHO AM I?" is the right title for the picker. Don't translate every word but do route the brand-load-bearing strings through TBOI's voice.
4. **Glanceable status, dense detail** — status pills use 3-letter codes (REST/FIGHT/CLEAR) at 18px+ so a half-second peek is enough. Detail panels and chat content use 14px monospace and pack information densely. No bloated card padding, no centered-with-tons-of-whitespace layouts.
5. **Trust the existing token system** — the upstream pokegents codebase had a thorough theme-token architecture (~150 tokens across themes.json + tboi.css). The TBOI rebrand re-uses every token; it never invents parallel hex values. When tempted to add a token, first check if there's an existing one that fits.

## Tech Stack

- React 19 + Vite 6 + TypeScript 5.7 + Tailwind 3.4 (frontend)
- Go 1.22+ (backend, file watcher, websocket, Claude Code hooks)
- modernc.org/sqlite (pure-Go SQLite — no CGO, search works in release binary)
- `image-rendering: pixelated` on all sprite containers
- Theme tokens come from `dashboard/web/src/theme/themes.json` (runtime source of truth) and `dashboard/web/src/styles/tboi.css` (pre-paint defaults + class-scoped rules)
